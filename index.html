Approach · i — iii
i.

Read before you design

Every page is a piece of writing before it is a layout. I draft the headline, the sub-headline, and the body first; the grid follows the cadence of the sentences, never the other way around. A page that hasn't been read aloud isn't finished.

ii.

Restraint is a feature

One type family. One column measure. One accent, if any. The web is loud by default; the work is in choosing what to leave out so the remaining elements can be heard. White space is not emptiness — it is the page breathing.

iii.

Ship, then refine

Every project in this index is live. None are case studies of what could have been. The discipline of finishing — of pressing deploy, of letting a page exist in public — is itself half of the craft. The web rewards pages that ship.

Plate iv PORTRAIT, 2024

I build the web the way I'd rather read it — slowly, plainly, and with care.

I'm Sangeen Khan, a frontend developer based in Riyadh, Saudi Arabia. I work alone, mostly on small projects for people who care about how things read. The portfolio leans toward editorial — blogs, libraries, restaurants, travel journals — because those are the briefs that let typography do its work.

I write HTML and CSS by hand, keep JavaScript to the minimum a page actually needs, and prefer systems that can be maintained by one person on a slow afternoon. Every project linked above is deployed and in use; none are demos dressed up as case studies.

If you have a project in mind — a journal, a catalogue, a single product that deserves its own page — I'd be glad to hear about it.
